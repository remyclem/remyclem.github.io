Mémento Markdown

Les titres

  • # Titre : Titre 1
  • ## Titre : Titre 2
  • ### Titre : Titre 3
  • etc

Gras, italique, barré

  • **text** ou __text__ : en gras
  • *text* ou _text_ : en italique
  • ***text*** ou ___text___ : en gras et italique
  • ~~text~~ : barré

Sauter à la ligne, paragraphes

  • Pour sauter à la ligne, il faut terminer la ligne avec deux espaces.
  • Pour écrire un nouveau paragraphe, il faut insérer une ligne vide.
    Noter qu'une ligne avec un espace n'est pas une ligne vide pour l'interpréteur.

Lien hypertexte, ancre sommaire

  • Lien vers un projet github : [lien vers un projet github](https://github.com/remyclem/Iris_classification)
  • Utiliser du code HTML fonctionne aussi, par exemple voici un lien vers une vidéo youtube : 
    <a href="https://www.youtube.com/watch?v=3atXidqpuZs" target="_blank">
  <img src="https://img.youtube.com/vi/3atXidqpuZs/0.jpg" alt="" width="240" height="180" border="10"/>
</a>
  • Une ancre vers un des titre :
    [Let's go back to the top](#going-to-the-next-line)
    Les majuscules sont passées en minuscule et les espaces sont remplacés par des tirets.
  • Pour réaliser un sommaire, il suffit de commencer le fichier markdown par un ensemble de telles ancres.

Insérer une image

  • Il faut passer par du code HTML : 
    <p align="center">
<img src="../img/viking_tux.png">
</p>

Les puces

  • Pour une simple liste : 
    * Une
* simple
* liste
  • Pour une liste ordonnées : 
    1. une
2. liste
3. ordonnée

Citations

  • Se fait en commançant la ligne avec le caractère >. 
    > Le courage n'est pas l'absence de peur,
> mais la capacité de vaincre ce qui fait peur.
> _Nelson Mandela_

Mise en évidence de lignes de code

  • Il faut utiliser des guillemets inversées, ou backtick en anglais : ` `cd /tmp` devient cd /tmp

  • Pour de long blocs de code, on préférera utiliser les triples guillemets inversées : 
    ```bash
> cd tmp
> ls -l
```
